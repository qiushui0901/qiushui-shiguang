## 项目经历

### 「拾光」泛生活内容分享社区  
*2025年9月 - 2026年4月 · 核心工程师*

**技术栈：** Spring Cloud Alibaba、Spring AI、MySQL、Redis、RocketMQ、Elasticsearch、Canal、XXL-Job、Caffeine、Roaring Bitmap

**项目描述：**  
面向内容社区场景，设计并实现笔记发布、点赞、收藏、关注、评论、搜索、推荐及AI辅助创作等核心功能；围绕高并发互动写入、热点内容访问、搜索同步、推荐召回和异步一致性等场景进行性能优化与架构设计。

**高并发互动写入与一致性保障：**  
针对点赞、收藏、关注等高频操作，设计 **Redis 预写 + RocketMQ 异步落库**方案，通过 **Redis Lua 脚本**完成状态判断与原子更新，并结合顺序消息保证同一用户连续操作的消费顺序；消费者侧采用 Buffer Trigger 聚合批量写入，降低高峰期数据库写入压力。针对 Redis、MQ、MySQL 之间的异步链路，设计 **消息幂等、顺序消费、失败重试和补偿对齐**机制；基于 XXL-Job 分片广播实现低峰期增量对账，保证核心业务数据最终一致。

**多级缓存优化与接口优化：**  
设计 **Caffeine + Redis 二级缓存** 缓存热点笔记、用户信息和计数数据，并通过缓存失效通知、TTL 和延迟删除降低本地缓存与 Redis 的不一致窗口；使用 **CompletableFuture** 并行聚合笔记详情、用户信息和互动状态，在 benchmark 中将接口 RT 从串行 187ms 降至并行 62ms。

**个性化推荐系统：**  
设计 **热度、关注、话题** 三路召回架构，基于 ES 的 `function_score` 实现热度与时间衰减排序，通过消费点赞/收藏 MQ 事件构建用户兴趣画像，并使用 **Redis ZSet** 维护话题兴趣权重；在 5w 笔记、1w 用户的模拟数据集上进行 benchmark，三路召回覆盖率较单路热度召回提升 95.7 倍，不同用户 Feed 重叠率由 100% 降至 1.1%。

**AI Agent 辅助创作：**  
基于 **Spring AI** 设计内容理解、创作助手、内容审核、搜索总结等专项 Agent，通过 **Orchestrator** 编排不同 Agent 的调用流程，并结合 **Tool Calling** 接入 ES 检索、Redis 热度数据和敏感词检测能力；构建 L1 冒烟测试与 L2 Golden Dataset 评估集，对 Agent 路由、工具调用和输出格式进行稳定性验证，L2 测试通过率 94.3%。
